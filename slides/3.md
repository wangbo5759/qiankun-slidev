# 巨石应用的痛点

**Like Micro-Servie**

<br>

<v-clicks>

- 📝 **Spa-based** --> 子系统间直接耦合紧密



- 🎨 **Feature-Rich** --> <u>必须使用相同技术栈</u>，引入新技术难，合并其他子系统难



- 🧑‍💻 **Developer Seperate** --> 调试难度高，运维成本高



- 🤹 **Indivisual Git Repo** --> 受整体拖累，<u>启动运行性能低</u>



- ❎ **Indivisual Git Repo** --> 子系统崩溃，影响整个系统

</v-clicks>
